Configuring X11 Display Server for Windows-Subsystem-for-Linux-2

This guide follows https://www.how2shout.com/how-to/run-linux-gui-apps-on-windows-10-with-native-bash.html

If you have a Windows Subsystem for Linux 2 (WSL2) install on windows 10, you might someday want to run an app with GUI. If you have WSL 1, which was the standard until the May 2020 update, you can directly follow the guide on how2shout.com. However, because of the way version 2 handles networking, some modification of that guide is needed to get everything working. For step 1-5, you the instructions are the same as the guide. Install VcXsrv, and launch it.

Step 6 Alternative: Open XLaunch

When you open XLaunch, you need to check disable access control on extra settings. Otherwise the instructions are the same as the other guide. Step 7 can also be followed, saving it for running later. I personally save the conf file to my desktop, for easy future launches.

Step 8 Alternative: Display Server Constants in Bash

In order to get WSL2 to connect to the running display server on windows, it needs to know how to connect. The following command can be run manually each time you want to run a GUI application, or alternatively you can add it to the end of your ~/.bashrc or ~/.zshrc to have it automatically run when wsl2 starts up.

export DISPLAY=$(cat /etc/resolv.conf | grep nameserver | awk '{print $2; exit;}'):0.0

After this restart WSL2, and everything should work. You can test it by running gedit or any other gui application.
